The Challenge :: Hackathon Focus

THE PROBLEM WE SOLVED.

VenueIQ was built to directly tackle the three hardest physical challenges of large-scale sporting events.

01. Crowd Movement
Fans get bottlenecked. We implemented a section-aware routing engine that dynamically adapts its walking directions based on live capacity heatmaps, steering traffic away from congested concourses.
02. Waiting Times
Lines are completely unpredictable. Our Gemini assistant mathematically calculates the shortest combined walk-plus-wait time to save fans an average of 15+ minutes per trip.
03. Live Coordination
Phones fail in crowded stadiums. VenueIQ tracks group locations instantly, pushes active congestion alerts, and falls back to a locally cached offline engine when Wi-Fi/5G drops.
🏟️
Live Simulation Architecture
Because real stadium security sensors are proprietary, this demo uses a Live Pseudo-Random Mock Data Engine configured for Wembley Stadium. A headless Node.js scheduler actively manipulates 8 zones (gates, food courts, and restrooms) every 2 minutes. This creates genuine, fluctuating capacity hotspots — forcing the Gemini AI to dynamically reason and react to changing congestion states exactly as it would in production.

Features

BUILT FOR
REAL EVENTS.

Every feature is designed around what fans actually need at a live sporting event — not what looks good in a pitch deck.

F-01
Context-Aware AI Chat
Gemini receives live zone data alongside every message. Ask "shortest food queue?" — get a specific answer: "West Food Court, 6 min, saves ~8 min vs East."
Gemini 1.5 Flash
F-02
Live Crowd Heatmap
Canvas-rendered stadium aerial view with radial gradient density blobs. Each zone shows its wait time printed live, sized by capacity level.
Canvas API
F-03
Real-Time Zone Cards
8 live zone cards with animated fill bars and color-coded wait times — green under 5 min, amber to 15, red above. Updates every 2 minutes.
Firebase RTDB
F-04
Proactive AI Alerts
No more manually checking the app. Gemini automatically scans all zones every 5 minutes and pushes natural-language alerts when it detects congestion.
Scheduled Functions
F-05
Offline Fallback
Intent-matching local engine handles all core queries even with no network. Works perfectly without Firebase config, Gemini key, or internet — never fails fans.
PWA + SW Cache
F-06
Crowd Reporting
Fans tap to report congestion at any zone. Cloud Functions aggregate the last 20 crowd reports to dynamically update that zone's capacity and wait time.
Crowdsourced Intel
F-07
Multimodal Vision
Fans can take a photo of a queue. Gemini 2.0 Flash analyzes the image, identifies the density, and provides a real-time wait estimation in seconds.
Gemini 2.0 Flash
F-08
Voice Accessibility
Hold to speak using the Web Speech API. Native text-to-speech engine speaks the AI's response aloud, allowing hands-free navigation while walking.
Voice UX
F-09
Operator Dashboard
A God-mode Mission Control dashboard for stadium staff. Monitor live heatmaps, trigger automated staggered exits, and broadcast emergency alerts to all fans.
Real-Time Control
